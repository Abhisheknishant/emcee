lnprobfn(coords, *args)¶

This method computes the natural logarithm of the posterior probability up to a constant. User shouldn’t usually need to touch this implementation. Override the lnpriorfn() and lnlikefn() methods instead.

Parameters:coords – The coordinates where the probability should be evaluated.
new()¶

Return a new instance of the walker. The default implementation is similar to a shallow copy after removing the “stateful” attributes. These are attributes that always depend on the coordinates (things like the ln-probability) so you wouldn’t want to copy them to a new instance. This gets a tiny performance improvement over direct use of copy.copy but we’ll call it a lot of times so maybe it’s worth it. You might want to implement a more efficient copy function if you know more about your class. If your model has other stateful attributes, you can also just append them to the class-level stateful_attributes list.

propose(coords)¶

This function is called when a proposal is updating the walker coordinates.

Parameters:coords – The new coordinates of the walker.
class emcee.SimpleWalker(lnprior_or_lnprob, lnlikefn=None)¶

A simple extension of the BaseWalker subclass that accepts functions that evaluate the ln-prior and ln-likelihood functions.

Parameters:
  • coords – The initial coordinate vector for the walker.
  • lnprior_or_lnprob – A callable that evaluates the ln-prior at a given coordinate vector. If lnlikefn isn’t given, this function is assumed to evaluate the log-probability up to a constant.
  • lnlikefn – (optional) A callable that evaluates the ln-likelihood at a given coordinate vector.

Sampling¶

class emcee.Ensemble(walker, coords, *args, **kwargs)¶

An Ensemble is a set of walkers (usually subclasses of BaseWalker).

Parameters:
  • walker – The walker class.
  • coords – The 2-D array of walker coordinate vectors. The shape of this array should be (nwalkers, ndim).
  • pool – (optional) A pool object that exposes a map function. This is especially useful for parallelization.
  • random – (optional) A numpy-compatible random number generator. By default, this will be the built-in numpy.random module but if you want the ensemble to own its own state, you can supply an instance of numpy.random.RandomState.

Note

Any extra arguments or keyword arguments are pass along to the walker initialization.

coords¶

The coordinate vectors of the walkers.

lnlike¶

The ln-likelihoods of the walkers up to a constant.

lnprior¶

The ln-priors of the walkers up to a constant.

lnprob¶

The ln-probabilities of the walker up to a constant.

propose(coords, slice=slice(None, None, None))¶

Given a new set of coordinates return a list of new walkers evaluated at these locations.

Parameters:
  • coords – The new coordinate matrix. It should be (nwalkers, ndim).
  • slice – (optional) The update can optionally only be applied to a subset of the walkers. This should be a slice object that can be used to index the walkers list.
update()¶

Update the coordinate matrix and probability containers given the current list of walkers. Moves should call this after proposing and accepting the walkers.

class emcee.Sampler(schedule=None, backend=None)¶
reset()¶

Clear the chain and reset it to its default state.

run(ensemble, niter, **kwargs)¶

Starting from a given ensemble, run a specific number of steps of MCMC. In practice, this method just calls Sampler.sample() and and returns the final ensemble from the iterator.

Parameters:
  • ensemble – The starting Ensemble.
  • niter – The number of steps to run.
  • store – (optional) If True, save the chain using the backend. If False, reset the backend and don’t store anything.
sample(ensemble, niter=None, store=None)¶

Starting from a given ensemble, start sampling as an iterator yielding each updated ensemble.

Parameters:
  • ensemble – The starting Ensemble.
  • niter – (optional) The number of steps to run. If not given, the iterator will run forever.
  • store – (optional) If True, save the chain using the backend. If False, reset the backend and don’t store anything.

class emcee.moves.DEMove(sigma, gamma0=None, **kwargs)¶

A differential evolution proposal implemented following Nelson et al. (2013).

Parameters:
  • sigma – The standard deviation of the Gaussian used to stretch the proposal vector.
  • gamma0 – (optional) The mean stretch factor for the proposal vector. By default, it is \(2.38 / \sqrt{2\,\mathrm{ndim}}\) as recommended by MAGIC and the two references.

class emcee.moves.MHMove(proposal_function, ndim=None)¶

A general Metropolis-Hastings proposal.

Parameters:
  • proposal – The proposal function. It should take 2 arguments: a numpy-compatible random number generator and a (K, ndim) list of coordinate vectors. This function should return the proposed position and the log-ratio of the proposal probabilities (\(\ln q(x;\,x^\prime) - \ln q(x^\prime;\,x)\) where \(x^\prime\) is the proposed coordinate).
  • ndim – (optional) If this proposal is only valid for a specific dimension of parameter space, set that here.
